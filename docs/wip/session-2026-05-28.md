## Decisions

**ADR-0006: Mirror Node parsing lives in `core/`, takes raw text, preserves amounts as strings.**
Mirror Node returns balances as unquoted JSON numbers. `JSON.parse` coerces them through
IEEE-754 doubles before any Zod schema sees the value — precision is silently gone. The fix is
a `JSON.parse` reviver that captures designated amount fields (`total_supply`, `initial_supply`,
`max_supply`, `balance`) from the raw source-text literal, keeping them as exact strings. This
requires Node ≥ 22 (the reviver `context.source` third argument shipped in V8 12.1 / Node 21.7+).
Node floor raised from ≥18 to ≥22 in `package.json` `engines`. A startup probe throws loudly
if the runtime does not support source-text access; silent fallback to a lossy parse was
explicitly rejected. See `/home/kb/Desktop/projects/hedera-native-rwa-dapp/docs/adr/0006-bigint-safe-mirror-node-parsing.md`.

**ADR-0007: `deployments.json` is gitignored; `deployments.example.json` is the committed template.**
Runtime reads token/topic/operator ids from `deployments.json`. The issuer CLI scripts write
to it after each creation step. Gitignored to prevent id confusion across developer environments.
See `/home/kb/Desktop/projects/hedera-native-rwa-dapp/docs/adr/0007-deployments-json-and-issuer-script-runtime.md`.

**ADR-0008: WalletConnect 1.x (`@hashgraph/hedera-wallet-connect` ^1.3.4) accepted for v1.**
The library carries deprecated WalletConnect-v1 transitive dependencies (`npm audit` reports
advisories). Accepted for testnet teaching use only. The revisit trigger is documented: upgrade
when `@hashgraph/hedera-wallet-connect` ^2.x ships with stable WalletConnect-v2 internals.
See `/home/kb/Desktop/projects/hedera-native-rwa-dapp/docs/adr/0008-frontend-wallet-integration.md`.

**ADR-0009: Frontend consumes root package via `@core` alias pointing to `../dist/`.**
The frontend does not import TypeScript source from `src/core/` directly. The root package
must be built first (`npm run build` from root) — `vite.config.ts` `prebuild`/`predev` hooks
handle this. This avoids dual-package bundling issues and keeps `core/` free of any browser
assumptions. See `/home/kb/Desktop/projects/hedera-native-rwa-dapp/docs/adr/0009-frontend-core-integration-and-read-model.md`.

**ADR-0010: Demo mode + Playwright for screenshot capture; placeholder PNGs committed.**
Wallet-gated screenshots cannot be reliably captured against a real HashPack browser extension
in CI or on a machine without the extension. A dev-only `VITE_DEMO=1` mode provides a stub
signer and canned outcomes so Playwright can capture all eight views. Read-only panels stay
real Mirror data. The demo branch is clearly fenced and unreachable without the flag. Placeholder
PNGs are committed so the README renders on GitHub before real capture happens.
See `/home/kb/Desktop/projects/hedera-native-rwa-dapp/docs/adr/0010-screenshots-and-banner-capture.md`.

**`sdk/` never calls `JSON.parse` on Mirror responses (implicit convention, not its own ADR).**
The pattern is: `fetch(...).text()` → pass raw text to `core/` parser. Every `sdk/operations/`
file follows this. The core-purity guard enforces the inverse (nothing in `core/` touches
`@hashgraph/sdk`), but the mirror-side convention is only tested by the integration smoke.
If someone adds a new Mirror operation to `sdk/`, they must follow `fetch().text()` → core.

## Watch Out For

**PATH in Bash sessions.** The Bash tool's shells do not source `~/.bashrc` or `~/.profile`.
Node 22 was installed via nvm at `~/.nvm/versions/node/v22.22.3/bin`. Every `node`/`npm`/`npx`
command must be prefixed with:
```
export PATH="$HOME/.nvm/versions/node/v22.22.3/bin:$PATH"
```
Without this, every npm/node command silently fails with "command not found."

**No `deployments.json` or `.env` in this environment.** There is no live testnet deployment
here. The integration test skips cleanly without credentials (`test/integration/`), but the
issuer scripts and the frontend `npm run screenshots` will not work without these files.

**Frontend build depends on root build output.** `frontend/vite.config.ts` resolves `@core`
to `../dist/`. If root `dist/` is stale or missing, the frontend build will fail with a module
resolution error. The `predev`/`prebuild` hooks in `frontend/package.json` run `npm --prefix ..
run build` automatically, but this requires the root package to typecheck cleanly first.

**The `VITE_DEMO=1` branch in `frontend/src/actions/associate.ts` and `transfer.ts` is
dev-only and must not be shipped.** It is gated on `import.meta.env.VITE_DEMO === '1'`, which
Vite tree-shakes out of production builds. If the env var is accidentally set in a production
build, the canned outcomes will be served instead of real transactions — a silent, dangerous
lie. Never set `VITE_DEMO` in a production or staging `.env`.

**The `amountKeys` set in `src/core/mirror/json.ts` is hand-maintained.** It currently covers
`total_supply`, `initial_supply`, `max_supply`, `balance`. If a new Mirror Node endpoint returns
a large-integer amount field not in this list, it will parse lossily (truncated by IEEE-754)
without any error. Any new Mirror parser added to `core/` must check whether its payload
contains large-integer fields and add them to `amountKeys` if so.

**`npm audit` will report advisories.** These come from `@hashgraph/hedera-wallet-connect` 1.x
transitive WalletConnect-v1 dependencies. They are accepted per ADR-0008 and are not blockers
for testnet teaching use. Do not attempt to resolve them by downgrading or patching the wallet
library without re-validating the full frontend wallet flow.

**Placeholder PNGs in `docs/images/` are synthetic.** They were generated by
`frontend/scripts/placeholders.mjs` and committed so the README renders on GitHub. They do not
show real token state. After `npm run screenshots` runs on the maintainer's machine and overwrites
them, commit the replacements before treating the repo as "v1 done."
